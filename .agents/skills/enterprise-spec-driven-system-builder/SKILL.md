# Enterprise Spec-Driven System Builder

Follow `AGENTS.md` and `.github/skills/enterprise-spec-driven-system-builder.yaml`.

Workflow:

1. Start from a product/use-case spec in `.github/specs/` or user-provided requirements.
2. Identify assumptions, open questions, functional requirements, NFRs, and acceptance criteria.
3. Plan backend, frontend, database, security, observability, tests, and docs as separate but aligned workstreams.
4. Validate against `.github/policies/` before implementation.
5. Keep decisions traceable from requirement to code, test, and documentation.

Use `.github/docs/orchestrator-workflow.yaml` for the reference workflow when detailed orchestration is needed.
