# Documentation Generation

Follow `AGENTS.md` and `.github/skills/documentation-generation.yaml`.

When documenting:

1. Document the current implemented behavior, not imagined behavior.
2. Include executable setup/test commands when useful.
3. Keep docs consistent with `.github/docs/`, `.github/specs/`, and existing ADRs.
4. Call out prerequisites, environment variables, Docker/Testcontainers needs, and known limitations.
